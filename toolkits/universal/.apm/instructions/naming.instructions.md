# Naming Things

- Names must be self-descriptive. Avoid puns, uncommon acronyms, version numbers, or brand names that obscure purpose.
- Use the same name consistently (e.g. a GitHub repo name matches its hostname).
- Prefer clear service names, for example: Peoplefinder, Send Money to a Prisoner, Help with Fees.
- Avoid ambiguous names and internal acronyms (e.g. CCCD, PVB2, DOMIS).

> Source: [Naming things](https://technical-guidance.service.justice.gov.uk/documentation/standards/naming-things.html)
