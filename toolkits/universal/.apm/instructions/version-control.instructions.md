# Version Control and Branching

- Use GitHub for all projects.
- Branching: `feature/*` for features, `develop` for integration, `main` for production-ready code.
- Commit messages: include a ticket/issue reference; use the format `type(scope): description` (e.g. `feat(auth): add JWT authentication`) in the imperative mood.
